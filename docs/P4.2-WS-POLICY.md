# P4.2 - WebSocket Protection Policy

> **Status:** Implemented  
> **Version:** 1.0  
> **Date:** 2026-04-08

---

## 1. Identity

### Priority (most reliable first)

1. **`userId`** - Authenticated identity via token
2. **`sessionId`** - Identifiable session (if available)
3. **`ipHash`** - Hashed client IP (fallback)

### Key Format

```
ws:{dimension}:{type}:{hash}

Examples:
  ws:conn:user:a1b2c3d4...
  ws:conn:ip:e5f6g7h8...
  ws:msg:user:a1b2c3d4...
  ws:reconn:session:x9y0z1...
```

---

## 2. Limits by Dimension

| Dimension | Limit | Window | Behavior on Exceed |
|-----------|-------|--------|-------------------|
| **Connection** | 10 | 60s | Reject code 1013 (TRY_LATER) with Retry-After |
| **Message** | 50 | 60s | Close code 1008 (POLICY_VIOLATION) with quota info |
| **Reconnect** | 5 | 60s | Forced 5s delay + reject 1013 |

### Separate Counters

Each dimension has its own counter and window:
- `ws:conn:{id}` - Connection counter
- `ws:msg:{id}` - Message counter  
- `ws:reconn:{id}` - Reconnect counter

## 3. Blocking Behavior

### Code 1013 - TRY_LATER (Connection/Reconnect)

```json
{
  "code": 1013,
  "reason": "WS_QUOTA_EXHAUSTED",
  "retryAfter": 45,
  "limit": 10,
  "window": "60s"
}
```

### Code 1008 - POLICY_VIOLATION (Message)

```json
{
  "code": 1008,
  "reason": "WS_MESSAGE_QUOTA_EXCEEDED",
  "quota": {
    "limit": 50,
    "used": 51,
    "resetAt": "2026-04-08T12:00:00Z"
  }
}
```

## 4. Observability

### Aggregated Metrics (no explosive labels)

```
ws_connection_limit_hits_total    // Counter for connection limit blocks
ws_message_limit_hits_total       // Counter for message limit blocks
ws_reconnect_penalty_hits_total   // Counter for reconnect penalties
ws_active_connections             // Gauge of active connections
ws_blocked_connections            // Gauge of blocked connections in window
```

### Structured Logging

```json
{
  "timestamp": "2026-04-08T12:00:00Z",
  "level": "WARN",
  "event": "WS_RATE_LIMIT_HIT",
  "dimension": "connection|message|reconnect",
  "identityType": "user|session|ip",
  "identityHash": "a1b2c3...",
  "limit": 10,
  "current": 11,
  "windowMs": 60000,
  "action": "REJECTED"
}
```

**Note:** Never log raw identifiers (userId, IP, sessionId in plain text).

## 6. Quick Runbook

### Client Receives 1013
1. Wait `retryAfter` seconds
2. Attempt reconnect with exponential backoff
3. If persists > 3 attempts, notify user

### Client Receives 1008
1. Reduce message send rate
2. Wait for next window (60s)
3. Continue normally

### Admin Sees Alert
1. Check `ws_blocked_connections`
2. If > 100 blocks/min, investigate attack
3. Possible action: temporarily increase limit
