## 1. Blocking Behavior

### 1.1 Code 1013 - TRY_LATER

**When it occurs:**
- Connection limit exceeded (> 10 in 60s)
- Reconnect limit exceeded (> 5 in 60s)

**Client response:**
```json
{
  "code": 1013,
  "reason": "{\"code\":\"WS_QUOTA_EXHAUSTED\",\"retryAfter\":45,\"limit\":10}",
  "retryAfter": 45
}
```

**Client action:**
1. Wait `retryAfter` seconds
2. Attempt reconnect with exponential backoff
3. If fails 3x consecutively, notify user

**Client code example:**
```javascript
let attempts = 0;
const maxAttempts = 3;

function connectWithBackoff(url) {
  const ws = new WebSocket(url);
  
  ws.onclose = (event) => {
    if (event.code === 1013) {
      const reason = JSON.parse(event.reason);
      const delay = reason.retryAfter * 1000;
      
      attempts++;
      if (attempts < maxAttempts) {
        setTimeout(() => connectWithBackoff(url), delay);
      } else {
        console.error('Max retry attempts reached');
      }
    }
  };
}
```

### 1.2 Code 1008 - POLICY_VIOLATION

**When it occurs:**
- Connection without token
- Invalid token
- Messages exceeding quota (> 50 in 60s)

**Client response:**
```json
{
  "code": 1008,
  "reason": "Authentication required"
}
```

**Client action:**
1. Check credentials
2. Reauthenticate if necessary
3. Reduce message rate

## 3. Troubleshooting

### 3.1 Legitimate User Blocked

**Symptom:** User reports "cannot connect"

**Diagnosis:**
```bash
# Check server logs
grep "WS_RATE_LIMIT_HIT" /var/log/ctrlclaw/ws.log | grep <user-hash>

# Check current metrics
curl http://localhost:3000/api/metrics | grep ws_
```

**Actions:**
1. Check if user actually exceeded limits
2. If DDoS attack: add IP to blacklist
3. If false positive: temporarily increase limit

---

### 3.2 Reconnect Storm

**Symptom:** Metric `ws_reconnect_penalty_hits` spikes

**Diagnosis:**
```bash
# Identify pattern
grep "WS_RATE_LIMIT_HIT" /var/log/ctrlclaw/ws.log | grep "dimension.*reconnect" | head -20
```

**Actions:**
1. Identify origin (IP or user)
2. If malicious: block at firewall
3. If client bug: notify frontend team

---

### 3.3 Server Overloaded

**Symptom:** `ws_active_connections` near limit

**Actions:**
1. Check memory: `free -h`
2. Check CPU: `top`
3. If needed: graceful server restart

  -H "Connection: Upgrade" \

```bash

kill -SIGINT <pid>

node scripts/start-ws-server.js
```
